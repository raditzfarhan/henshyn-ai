# Mission Breakdown — Task Decomposition

## Overview
Decompose a feature or project into concrete, sized, prioritized development tasks. Every task must be independently completable with a verifiable done state.

## Process
1. **Identify epics** — what are the 2–6 major feature areas? Name them in 2–6 words each.
2. **Break into tasks** — for each epic, generate tasks that are 1–4 hours of work each. If a task is bigger, split it.
3. **Write acceptance criteria** — for each task: "done when X". Must be verifiable, not vague.
4. **Mark dependencies** — which tasks must complete before others can start?
5. **Prioritize** — rank each task: High (blocks other work or users), Med (important but not blocking), Low (nice to have).

## Output Format
Follow the task-manager agent output structure exactly.
