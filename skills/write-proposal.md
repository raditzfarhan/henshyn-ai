# Skill: Write Proposal

## Purpose
Structure a project into a clear, persuasive proposal that stakeholders can act on.

## Process
1. **Clarify the core goal** — write it in one sentence. If you can't, the goal isn't clear yet.
2. **Define scope boundaries** — what's explicitly in and out. Out-of-scope is as important as in-scope.
3. **Identify stakeholders** — who needs to approve, build, or use this?
4. **Propose tech stack** — pick the right tool for the constraints (timeline, team, budget). Explain why.
5. **Estimate phases** — break into 2–4 phases. Each phase delivers something working.
6. **Apply cost-breakdown skill** — after phases are defined, calculate development cost (hours × rate per phase), one-time setup costs, monthly operational costs, and optional add-ons. This is mandatory for client-facing proposals.

## Output Format
Follow the proposal-writer agent output structure exactly.
